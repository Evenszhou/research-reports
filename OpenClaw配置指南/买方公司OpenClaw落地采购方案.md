## 四、OpenClaw 配置要点

### 4.1 核心配置清单

```yaml
# ~/.openclaw/config.yaml 关键配置

gateway:
  host: 0.0.0.0
  port: 19001
  # 建议Tailscale固定域名
  public_url: https://mac-mini.tailxxxx.ts.net:19001

channels:
  feishu:
    enabled: true
    appId: cli_xxxxxx
    appSecret: xxx
    encryptKey: xxx
    verificationToken: xxx
    # 回调地址配置在飞书开放平台
    
  dingtalk:
    enabled: false  # 如需钉钉可开启
    
models:
  # 主模型：DeepSeek
  default:
    provider: deepseek
    api_key: ${DEEPSEEK_API_KEY}
    model: deepseek-chat  # V3
    
  # 推理任务专用
  reasoning:
    provider: deepseek
    api_key: ${DEEPSEEK_API_KEY}
    model: deepseek-reasoner  # R1
    
  # 长文本专用
  long_context:
    provider: moonshot
    api_key: ${KIMI_API_KEY}
    model: kimi-k2-5
    
  # 代码任务专用
  coding:
    provider: anthropic
    api_key: ${CLAUDE_API_KEY}
    model: claude-3-5-sonnet-20241022

tools:
  enabled:
    - browser      # 浏览器自动化
    - filesystem   # 文件操作
    - python       # Python执行
    - sqlite       # 本地数据库
    - mcp:
        - fetch    # 网页抓取
        - git      # Git操作
```

### 4.2 飞书机器人配置流程

1. **创建飞书应用**
   - 访问 https://open.feishu.cn
   - 创建企业自建应用
   - 记录 App ID / App Secret

2. **配置权限**
   ```
   im:message
   im:message:send_as_bot
   im:chat
   im:chat:readonly
   contact:user.base:readonly
   ```

3. **配置事件订阅**
   ```
   请求网址: https://你的域名/webhook/feishu
   订阅事件: im.message.receive_v1
   ```

4. **发布并添加到企业**

### 4.3 Tailscale 组网（推荐）

```bash
# 在Mac Mini上安装
brew install tailscale
sudo tailscale up

# 查看分配的域名
tailscale status
# 输出: 100.x.y.z  mac-mini  your-email@domain.com

# 在飞书配置回调地址
https://mac-mini.tailxxxx.ts.net:19001/webhook/feishu
```

**优点**：无需公网IP、搬家不换地址、端到端加密

## 五、典型使用场景配置

### 场景1：研报摘要助手

```bash
# 在飞书群聊中@机器人
@投研助手 总结一下这份研报的要点
[上传PDF]

# OpenClaw自动处理：
# 1. 下载PDF → 2. 调用Kimi长文本分析 → 3. 返回结构化摘要
```

**配置**：
```yaml
skills:
  report_summary:
    trigger: ["总结", "摘要", "要点"]
    model: kimi-k2-5
    prompt: |
      请总结这份投研报告的核心观点：
      1. 投资评级与目标价
      2. 关键假设与逻辑
      3. 风险提示
      4. 核心数据表格
```

### 场景2：代码回测助手

```bash
# 在飞书中发送
@投研助手 帮我写个双均线策略的回测代码
股票池：沪深300
周期：日线
参数：短均线20日，长均线60日

# OpenClaw自动：
# 1. 生成vnpy/Backtrader代码 → 2. 保存到指定目录 → 3. 返回代码文件
```

**配置**：
```yaml
skills:
  backtest_code:
    trigger: ["回测", "策略", "代码"]
    model: claude-3-5-sonnet
    template: backtest_template.py
    output_dir: ~/strategies/
```

### 场景3：数据自动下载

```bash
# 定时任务配置
@投研助手 每天早上8点下载昨夜美股行情

# OpenClaw自动：
# 1. 访问Yahoo Finance → 2. 抓取数据 → 3. 保存CSV → 4. 发送到飞书
```

**配置**：
```yaml
cron:
  - name: us_market_data
    schedule: "0 8 * * 1-5"
    action: |
      fetch https://finance.yahoo.com/markets/stocks/most-active/
      extract table[data-test="most-active"]
      save ~/data/us_market_$(date +%Y%m%d).csv
      notify 飞书群 "美股数据已更新"
```

## 九、关键决策点

### 要不要本地部署大模型？

| 场景 | 建议 |
|------|------|
| 处理敏感数据（持仓/交易记录） | ✅ 本地部署 Qwen2.5-32B |
| 纯公开信息分析（研报/新闻） | ❌ API 足够，更便宜 |
| 高频调用（>10万次/天） | ✅ 本地部署更省成本 |
| 低频调用（<1万次/天） | ❌ API 更灵活 |

### 要不要双机热备？

- **初期**：单台 Mac Mini + 配置备份即可
- **稳定期**：双机热备，故障自动切换
- **备份命令**：`openclaw backup create --schedule daily`
