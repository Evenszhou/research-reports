## 飞书通道配置原理

### 架构图

```
┌─────────────┐         ┌──────────────────┐         ┌─────────────┐
│  飞书服务器  │ ──────→ │  OpenClaw Gateway │ ←─────→ │   Mac mini  │
│  (云端)     │  回调    │   (本地/服务器)    │  消息    │  (本地)     │
└─────────────┘         └──────────────────┘         └─────────────┘
       ↑                        ↑
       │                        │
   用户发消息              处理并回复
```

### 关键配置项

| 配置项 | 存储位置 | 说明 |
|--------|----------|------|
| App ID | `~/.openclaw/config.json` | 飞书应用唯一标识 |
| App Secret | `~/.openclaw/config.json` | 飞书应用密钥 |
| Encrypt Key | `~/.openclaw/config.json` | 消息加密密钥 |
| Verification Token | `~/.openclaw/config.json` | 事件验证令牌 |
| 回调地址 | 飞书开放平台 | Gateway 的公网访问地址 |

## 初始配置步骤

### 1. 创建飞书应用

1. 登录 [飞书开放平台](https://open.feishu.cn)
2. 创建企业自建应用
3. 记录以下信息：
   - App ID
   - App Secret

### 2. 配置权限

在飞书开放平台，为应用添加以下权限：

**消息权限：**
- `im:message` - 获取与发送消息
- `im:message:send_as_bot` - 以应用身份发消息
- `im:chat` - 获取群组信息
- `im:chat:readonly` - 读取群组信息

**用户权限：**
- `contact:user.base:readonly` - 获取用户基本信息

### 3. 配置事件订阅

1. 在应用管理 → 事件订阅
2. 配置请求网址：
   ```
   https://你的域名/webhook/feishu
   ```
3. 添加事件：
   - `im.message.receive_v1` - 接收消息

### 4. OpenClaw 配置

```bash
# 方式1：交互式配置
openclaw configure

# 方式2：命令行配置
openclaw config set channels.feishu.enabled true
openclaw config set channels.feishu.appId "cli_xxx"
openclaw config set channels.feishu.appSecret "xxx"
openclaw config set channels.feishu.encryptKey "xxx"
openclaw config set channels.feishu.verificationToken "xxx"

# 重启 Gateway
openclaw gateway restart
```

### 5. 发布应用

1. 在飞书开放平台提交审核
2. 审核通过后发布
3. 将应用添加到企业

## 搬家/断电后恢复

### 配置丢失情况分析

| 配置类型 | 存储位置 | 是否丢失 | 恢复难度 |
|----------|----------|----------|----------|
| App ID / Secret | `~/.openclaw/config.json` | ❌ 不丢失 | — |
| Encrypt Key / Token | `~/.openclaw/config.json` | ❌ 不丢失 | — |
| Gateway 状态 | 内存 | ⚠️ 需重启 | 简单 |
| 回调地址 | 飞书开放平台 | ❌ 不丢失 | — |
| 公网 IP / 域名 | 网络环境 | ⚠️ 可能变化 | 需更新 |

### 恢复流程

**步骤1：启动 Gateway**

```bash
# Mac mini 开机后，通常自动启动
# 如果没有，手动启动：
openclaw gateway start

# 检查状态
openclaw gateway status
```

**步骤2：检查网络**

```bash
# 查看公网 IP
curl ifconfig.me

# 查看当前 Gateway 监听地址
openclaw config get gateway.host
openclaw config get gateway.port
```

**步骤3：更新回调地址（如果 IP 变了）**

1. 登录 [飞书开放平台](https://open.feishu.cn)
2. 进入应用 → 事件订阅
3. 更新请求网址：
   ```
   https://新IP或域名/webhook/feishu
   ```

**步骤4：测试连接**

```bash
# 健康检查
openclaw doctor --fix

# 发送测试消息
# 在飞书给机器人发送消息，观察是否有响应
```

### 快速恢复脚本

```bash
#!/bin/bash
# feishu-recovery.sh

echo "🔍 检查 Gateway 状态..."
openclaw gateway status || openclaw gateway start

echo "🌐 获取当前公网 IP..."
CURRENT_IP=$(curl -s ifconfig.me)
echo "当前 IP: $CURRENT_IP"

echo "📋 检查飞书配置..."
openclaw config get channels.feishu.enabled
openclaw config get channels.feishu.appId

echo "🔧 执行健康检查..."
openclaw doctor --fix

echo "✅ 恢复完成，请发送测试消息验证"
```

## 常见问题排查

### 问题1：发送消息无响应

**排查步骤：**

```bash
# 1. 检查 Gateway 是否运行
openclaw gateway status

# 2. 检查飞书配置
openclaw config get channels.feishu

# 3. 查看 Gateway 日志
openclaw logs --tail 100

# 4. 检查防火墙
# 确保端口（默认 19001）可访问
```

### 问题2：回调地址验证失败

**原因：**
- 回调地址不可访问
- 域名解析错误
- 证书问题（HTTPS）

**解决：**

```bash
# 测试回调地址是否可访问
curl -v https://你的域名/webhook/feishu

# 如果使用 HTTPS，确保证书有效
openssl s_client -connect 你的域名:443
```

### 问题3：消息解密失败

**原因：**
- Encrypt Key 配置错误

**解决：**

```bash
# 重新配置 Encrypt Key
openclaw config set channels.feishu.encryptKey "正确的Key"

# 重启 Gateway
openclaw gateway restart
```

### 问题4：权限不足

**原因：**
- 飞书应用权限未配置
- 应用未发布或未添加到企业

**解决：**
1. 检查飞书开放平台的权限配置
2. 确保应用已发布
3. 确保应用已添加到企业通讯录

## 最佳实践

### 1. 使用固定域名（推荐）

**方案A：Tailscale（最简单）**

```bash
# 安装 Tailscale
brew install tailscale

# 启动并登录
sudo tailscale up

# 查看分配的域名
tailscale status
# 输出类似：100.x.y.z  mac-mini  your-email@domain.com  -

# 在飞书配置回调地址
# https://100.x.y.z:19001/webhook/feishu
# 或使用 Tailscale 提供的 MagicDNS
# https://mac-mini.tailxxxx.ts.net:19001/webhook/feishu
```

**优点：**
- 不管在哪，域名不变
- 内网穿透，无需公网 IP
- 免费且稳定

**方案B：ngrok**

```bash
# 安装 ngrok
brew install ngrok

# 启动隧道
ngrok http 19001

# 使用 ngrok 提供的域名
# https://xxx.ngrok.io/webhook/feishu
```

**缺点：**每次重启域名会变（付费版可固定）

### 2. 配置备份

```bash
# 定期备份 OpenClaw 配置
openclaw backup create --note "定期备份"

# 或手动备份
tar -czf openclaw-backup-$(date +%Y%m%d).tar.gz ~/.openclaw

# 备份到云端（如 GitHub 私有仓库）
# 注意：不要提交敏感信息到公开仓库！
```

### 3. 监控告警

```bash
# 添加心跳检测任务
openclaw cron add \
  --name "feishu-health-check" \
  --every "1h" \
  --message "检查飞书通道状态：openclaw doctor" \
  --channel feishu
```
