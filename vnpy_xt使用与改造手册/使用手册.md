## 框架概述

### 定位与功能

vnpy_xt 是 VeighNa 量化交易框架的数据服务接口，专门对接**迅投QMT（Quantitative Trading Platform）**。它提供：

1. **实时行情订阅** - 支持股票、期货、期权的实时Tick推送
2. **历史数据查询** - 获取K线（分钟/日线）和Tick级别的历史数据
3. **仿真交易** - 通过QMT客户端进行模拟下单、撤单
4. **账户查询** - 实时查询资金、持仓、委托、成交

### 支持的市场

| 市场类型 | 交易所 | 说明 |
|---------|--------|------|
| **股票/基金/债券** | SSE（上交所）、SZSE（深交所）、BSE（北交所） | A股、ETF、转债 |
| **期货/期权** | SHFE（上期所）、DCE（大商所）、CZCE（郑商所）、CFFEX（中金所）、INE（能源中心）、GFEX（广期所） | 商品期货、金融期货 |
| **ETF期权** | SSE、SZSE | 股票期权 |
| **期货期权** | 各期货交易所 | 商品期权、金融期权 |

## 架构设计

### 宏观架构

vnpy_xt 采用**分层架构**，遵循 VeighNa 的标准设计：

```
┌─────────────────────────────────────┐
│   VeighNa Trader (主应用)            │
├─────────────────────────────────────┤
│   XtGateway (交易网关)               │
│   ├─ XtMdApi (行情API)              │
│   └─ XtTdApi (交易API)              │
├─────────────────────────────────────┤
│   XtDatafeed (数据服务)              │
├─────────────────────────────────────┤
│   xtquant (迅投底层库)               │
│   ├─ xtdata (数据接口)              │
│   ├─ xttrader (交易接口)            │
│   └─ xtdatacenter (数据中心)        │
├─────────────────────────────────────┤
│   迅投QMT客户端 / 数据服务器          │
└─────────────────────────────────────┘
```

### 核心模块

#### 1. **XtGateway**（交易网关）
- **职责**：整合行情和交易功能，提供统一接口
- **关键功能**：
  - 连接管理（Token模式或客户端模式）
  - 行情订阅
  - 委托下单/撤单
  - 账户/持仓查询

#### 2. **XtMdApi**（行情API）
- **职责**：实时行情推送
- **关键功能**：
  - 合约信息查询
  - 行情订阅（订阅后持续推送Tick）
  - 行情数据解析（转换为VeighNa标准格式）

#### 3. **XtTdApi**（交易API）
- **职责**：仿真交易执行
- **关键功能**：
  - 下单/撤单
  - 委托/成交/持仓/资金查询
  - 交易回报推送

#### 4. **XtDatafeed**（数据服务）
- **职责**：历史数据查询
- **关键功能**：
  - K线历史查询（分钟/日线）
  - Tick历史查询
  - 数据格式转换（XT → VeighNa）

## 安装配置

### 1. 环境要求

- **Python**：3.10 / 3.11 / 3.12 / 3.13
- **操作系统**：Windows（迅投客户端仅支持Windows）
- **VeighNa**：4.0.0+ 版本（推荐使用 VeighNa Studio）

### 2. 安装步骤

#### 步骤1：安装 vnpy_xt

```bash
# 方式1：pip安装
pip install vnpy_xt

# 方式2：源码安装
git clone https://github.com/vnpy/vnpy_xt.git
cd vnpy_xt
pip install .
```

#### 步骤2：安装迅投 xtquant 库

**两种模式任选其一：**

**模式A：Token模式（仅数据服务）**
1. 申请试用账号：[VeighNa社区专属14天试用](https://xuntou.net/#/signup?utm_source=vnpy)
2. 从[迅投知识库](http://docs.thinktrader.net/)下载 xtquant 安装包
3. 解压后将 `xtquant` 文件夹放到 Python 的 `site-packages` 目录

**模式B：客户端模式（数据+交易）**
1. 安装迅投QMT客户端
2. 在客户端中点击【下载Python库】-【Python库下载】
3. 将下载路径中的 `xtquant` 文件夹拷贝到 Python 的 `site-packages`

### 3. 配置连接

在 VeighNa Trader 中配置：

#### Token模式（数据服务）
```json
{
  "datafeed.name": "xt",
  "datafeed.username": "token",
  "datafeed.password": "your_token_here"
}
```

#### 客户端模式（数据+交易）
```json
{
  "datafeed.name": "xt",
  "datafeed.username": "client",
  "datafeed.password": ""  // 留空
}
```

#### Gateway连接配置（交易网关）

在 VeighNa Trader 的网关配置界面填写：

| 参数 | 说明 | 示例 |
|------|------|------|
| **token** | 迅投Token | `your_token_here` |
| **股票市场** | 是否订阅股票 | 是/否 |
| **期货市场** | 是否订阅期货 | 是/否 |
| **期权市场** | 是否订阅期权 | 是/否 |
| **仿真交易** | 是否启用交易 | 是/否 |
| **账号类型** | 交易账号类型 | 股票/股票期权 |
| **QMT路径** | QMT安装路径 | `C:\\迅投QMT交易端` |
| **资金账号** | 资金账号 | `12345678` |

## 核心接口详解

### 一、数据服务接口（XtDatafeed）

#### 1. 初始化连接

```python
from vnpy_xt import Datafeed

# 创建数据服务实例
datafeed = Datafeed()

# 初始化（自动读取全局配置）
success = datafeed.init()
if success:
    print("数据服务连接成功")
else:
    print("连接失败")
```

**参数说明**：
- `output`：日志输出函数，默认 `print`
- **返回值**：`bool`（True=成功，False=失败）

**底层逻辑**：
- Token模式：调用 `xtdc.set_token()` 和 `xtdc.init()` 启动数据服务进程
- 客户端模式：直接使用本地客户端的数据服务

---

#### 2. 查询K线历史

```python
from datetime import datetime
from vnpy.trader.object import HistoryRequest
from vnpy.trader.constant import Exchange, Interval

# 构造查询请求
req = HistoryRequest(
    symbol="000001",           # 股票代码（不带交易所后缀）
    exchange=Exchange.SZSE,    # 交易所
    interval=Interval.DAILY,   # 时间周期：DAILY/MINUTE/TICK
    start=datetime(2024, 1, 1),
    end=datetime(2024, 12, 31)
)

# 查询K线
bars = datafeed.query_bar_history(req)

# 返回值：list[BarData]
for bar in bars[:5]:
    print(f"{bar.datetime}: 开{bar.open_price} 高{bar.high_price} 低{bar.low_price} 收{bar.close_price}")
```

**参数详解**：
- `symbol`：合约代码（如 `000001`、`IF2401`）
- `exchange`：交易所枚举（`Exchange.SSE`、`Exchange.SHFE` 等）
- `interval`：时间周期
  - `Interval.MINUTE`：分钟线
  - `Interval.DAILY`：日线
  - `Interval.TICK`：Tick数据（需用 `query_tick_history`）
- `start`：开始时间
- `end`：结束时间

**返回值**：`list[BarData]`，每个 BarData 包含：
- `datetime`：K线时间（K线开始时刻，北京时间）
- `open_price`：开盘价
- `high_price`：最高价
- `low_price`：最低价
- `close_price`：收盘价
- `volume`：成交量
- `turnover`：成交额
- `open_interest`：持仓量（期货专用）

**重要机制**：
1. **时间调整**：迅投的时间戳是K线结束时刻，vnpy_xt 自动转换为开始时刻
2. **集合竞价处理**：分钟线会自动合并9:25/9:29的竞价数据到开盘第一根K线
3. **涨跌停价**：通过合约信息查询获取

---

#### 3. 查询Tick历史

```python
# 查询Tick数据
req = HistoryRequest(
    symbol="000001",
    exchange=Exchange.SZSE,
    interval=Interval.TICK,
    start=datetime(2024, 1, 1, 9, 30),
    end=datetime(2024, 1, 1, 15, 0)
)

ticks = datafeed.query_tick_history(req)

# 返回值：list[TickData]
for tick in ticks[:5]:
    print(f"{tick.datetime}: 最新价{tick.last_price}, 量{tick.volume}")
    print(f"  买一: {tick.bid_price_1}({tick.bid_volume_1})")
    print(f"  卖一: {tick.ask_price_1}({tick.ask_volume_1})")
```

**返回值**：`list[TickData]`，每个 TickData 包含：
- `datetime`：时间戳
- `last_price`：最新价
- `volume`：成交量
- `turnover`：成交额
- `open_interest`：持仓量
- `bid_price_1` ~ `bid_price_5`：买一~买五价
- `ask_price_1` ~ `ask_price_5`：卖一~卖五价
- `bid_volume_1` ~ `bid_volume_5`：买一~买五量
- `ask_volume_1` ~ `ask_volume_5`：卖一~卖五量

---

### 二、交易网关接口（XtGateway）

#### 1. 连接网关

```python
from vnpy.event import EventEngine
from vnpy_xt import XtGateway

# 创建事件引擎
event_engine = EventEngine()

# 创建网关
gateway = XtGateway(event_engine, "XT")

# 配置参数
setting = {
    "token": "your_token",
    "股票市场": "是",
    "期货市场": "否",
    "期权市场": "否",
    "仿真交易": "是",
    "账号类型": "股票",
    "QMT路径": "C:\\迅投QMT交易端",
    "资金账号": "12345678"
}

# 连接（异步，会启动线程）
gateway.connect(setting)
```

**关键点**：
- 连接是**异步**的，会启动独立线程
- 需要先启动 QMT 客户端（客户端模式）
- 连接成功后会自动查询合约信息

---

#### 2. 订阅行情

```python
from vnpy.trader.object import SubscribeRequest

# 构造订阅请求
req = SubscribeRequest(
    symbol="000001",
    exchange=Exchange.SZSE
)

# 订阅
gateway.subscribe(req)

# 行情会通过事件引擎推送，需注册回调
def on_tick(event):
    tick = event.data
    print(f"{tick.symbol}: {tick.last_price}")

event_engine.register(EVENT_TICK, on_tick)
```

**参数**：
- `symbol`：合约代码
- `exchange`：交易所

**推送机制**：
- 订阅后，实时行情通过 `EVENT_TICK` 事件持续推送
- 每个Tick包含完整的盘口数据（5档买卖价）

---

#### 3. 委托下单

```python
from vnpy.trader.object import OrderRequest
from vnpy.trader.constant import Direction, Offset, OrderType

# 构造委托请求
req = OrderRequest(
    symbol="000001",
    exchange=Exchange.SZSE,
    direction=Direction.LONG,      # 方向：LONG/SHORT
    offset=Offset.NONE,            # 开平：NONE/OPEN/CLOSE
    type=OrderType.LIMIT,          # 类型：LIMIT（限价）
    price=10.5,
    volume=100,
    reference="my_strategy"        # 策略名（可选）
)

# 下单
orderid = gateway.send_order(req)

print(f"委托号: {orderid}")
```

**参数详解**：
- `direction`：
  - `Direction.LONG`：买入
  - `Direction.SHORT`：卖出
- `offset`：
  - `Offset.NONE`：现货交易（无开平概念）
  - `Offset.OPEN`：开仓（期货/期权）
  - `Offset.CLOSE`：平仓（期货/期权）
- `type`：
  - `OrderType.LIMIT`：限价单（当前仅支持限价）

**返回值**：
- `vt_orderid`：VeighNa统一委托号（格式：`gateway_name.orderid`）
- 空字符串表示委托失败

---

#### 4. 撤单

```python
from vnpy.trader.object import CancelRequest

# 构造撤单请求
req = CancelRequest(
    orderid="120231215143000000001",  # 本地委托号
    symbol="000001",
    exchange=Exchange.SZSE
)

# 撤单
gateway.cancel_order(req)
```

---

#### 5. 查询账户信息

```python
# 查询资金
gateway.query_account()

# 查询持仓
gateway.query_position()

# 数据会通过事件推送
def on_account(event):
    account = event.data
    print(f"账户: {account.accountid}")
    print(f"总资产: {account.balance}")
    print(f"可用资金: {account.available}")

def on_position(event):
    position = event.data
    print(f"合约: {position.symbol}")
    print(f"持仓: {position.volume}")
    print(f"成本: {position.price}")

event_engine.register(EVENT_ACCOUNT, on_account)
event_engine.register(EVENT_POSITION, on_position)
```

## 常见问题

### Q1：连接失败，提示"迅投研数据服务初始化失败"

**原因**：
1. xtquant 库未正确安装
2. Token 无效或过期
3. QMT 客户端未启动（客户端模式）

**解决**：
```python
# 检查 xtquant 是否安装
try:
    import xtquant
    print("xtquant 已安装")
except:
    print("请安装 xtquant")

# 检查 Token
# 登录 https://xuntou.net/#/userInfo 确认 Token 是否有效
```

---

### Q2：查询历史数据返回空列表

**原因**：
1. 时间范围不合理（结束时间早于开始时间）
2. 合约代码错误
3. 该时间段无数据（周末/节假日）

**解决**：
```python
# 检查合约代码格式
symbol = "000001"  # 不带交易所后缀
exchange = Exchange.SZSE

# 检查时间范围
start = datetime(2024, 1, 1)
end = datetime(2024, 12, 31)

# 查看日志输出
bars = datafeed.query_bar_history(req, output=print)
```

---

### Q3：订阅行情后没有推送

**原因**：
1. 未注册事件回调
2. 合约代码错误
3. 市场未开启（非交易时间）

**解决**：
```python
# 确保注册了回调
event_engine.register(EVENT_TICK, on_tick)

# 检查合约是否支持
# 先连接，等待合约信息推送（约10秒）
```

---

### Q4：下单失败，返回空字符串

**原因**：
1. 仿真交易未启用（配置中"仿真交易"="否"）
2. 合约不在支持范围（仅支持股票/ETF/期权）
3. 委托价格/数量不合法

**解决**：
```python
# 检查配置
setting["仿真交易"] = "是"

# 检查合约
if contract.exchange not in {Exchange.SSE, Exchange.SZSE, Exchange.BSE}:
    print("仅支持股票市场")

# 检查委托参数
price = 10.5   # 价格需 > 0
volume = 100   # 股票需为100的整数倍
```

---

### Q5：数据时间戳不对

**原因**：VeighNa 使用 K线开始时刻，迅投使用结束时刻

**说明**：
- vnpy_xt 已自动处理时间调整
- 分钟线：15:00 的K线表示 14:59-15:00 这根K线
- 日线：包含集合竞价数据

## 总结

vnpy_xt 作为 VeighNa 和迅投QMT的桥梁，提供了完整的量化交易能力：

1. **数据获取**：历史K线/Tick + 实时行情订阅
2. **交易执行**：下单/撤单/查询
3. **账户管理**：资金/持仓/委托/成交

**使用建议**：
- **数据服务**：使用 Token 模式即可，无需启动客户端
- **仿真交易**：必须启动 QMT 客户端并登录账号
- **生产环境**：建议使用迅投的实盘接口（需券商授权）
